# Brand & UI Guidelines

## 1. مصدر التصميم
يجب على المصمم ووكيل البرمجة استخدام ملفات PDF داخل:

```txt
assets/brand/
```

الملفات المتوقعة:

```txt
Banco Ricco Final Logo Big.pdf
BR Banco Ricco Final Logo.pdf
```

## 2. شخصية التصميم
- Premium.
- Damascus heritage.
- Coffee bank.
- Classic + modern.
- Elegant, not loud.

## 3. ألوان مبدئية

```css
:root {
  --br-black: #0D0D0D;
  --br-gold: #C9961A;
  --br-gold-dark: #9E7410;
  --br-cream: #F8F3E7;
  --br-white: #FFFFFF;
  --br-coffee: #4B2E1E;
  --br-espresso: #1F130E;
  --br-muted: #7A6A58;
  --br-success: #2E7D32;
  --br-warning: #B26A00;
  --br-danger: #B42318;
}
```

هذه ألوان أولية، ويجب تحسينها بعد مراجعة اللوغو داخل المشروع.

## 4. قواعد UX
- الصفحة الرئيسية لا تعرض كل المنتجات.
- أولوية العرض: الأكثر طلباً، اختيار المايسترو، الطلب السريع، البن، العروض.
- CTA واضح: اطلب للاستلام.
- لا تستخدم زخارف كثيرة على حساب القراءة.
- كل شاشة يجب أن تجيب: ماذا أفعل الآن؟

## 5. الواجهات الحرجة
- Splash.
- Home.
- Product Details.
- Coffee Bean Details.
- Grind Selection.
- Cart.
- Pickup Tracking.
- B.R Coins.
- Banco Card QR.
- Maestro Coffee Finder.

- Admin Product Editor.

## 6. ممنوعات
- لا تستخدم قالب مطعم جاهز.
- لا تجعل التطبيق متجراً عادياً.
- لا تجعل القصص طويلة.
- لا تخلط RTL وLTR.
- لا تعرض خيارات كثيرة في خطوة واحدة.
