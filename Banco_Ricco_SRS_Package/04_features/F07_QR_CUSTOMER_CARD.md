# F07 — QR Customer Card

## Phase
Phase 2

## الاسم التجاري
Banco Card

## الهدف
ربط تجربة العميل داخل التطبيق بالمحل عبر QR خاص.

## يظهر للعميل
```txt
اسم العميل
B.R Coins
QR Code
رسالة VIP
آخر/أفضل طلب لاحقاً
```

## يظهر للموظف بعد المسح
```txt
اسم العميل
رصيد B.R Coins
الطلبات الأخيرة
المشروب المفضل
العروض المتاحة
```

## قواعد الأمان
- QR لا يحتوي بيانات حساسة.
- QR يحتوي public token.
- يمكن إعادة توليده.
- لا تعرض رقم الهاتف كاملاً إلا بصلاحية.

## Acceptance Criteria
- كل عميل لديه QR.
- الموظف يستطيع المسح من لوحة التحكم.
- البيانات المعروضة تشغيلية وليست حساسة.
